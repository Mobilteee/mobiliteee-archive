Sketch2Face
简笔画人脸生成系统

基于条件生成对抗网络（cGAN）的端到端图像翻译

从一张手绘简笔画出发，让 AI 生成对应的逼真人脸照片。本项目为深度学习课程设计，基于 pix2pix 框架 + U-Net 生成器 + PatchGAN 判别器， 在 FS2K 数据集（2,104 对样本）上完成训练，并额外设计了两套改进方案：检索增强模块与数据增强微调策略。

阅读完整研究背景：从传统方法到 GAN 的范式转变

传统人脸素描合成方法依赖手工设计的特征提取算法（如边缘检测、稀疏表示），生成的人脸轮廓清晰度不足、纹理粗糙。这类方法的核心瓶颈在于：手工特征无法充分表征简笔画线条与真实人脸之间的复杂非线性映射关系。

2014 年 Goodfellow 提出生成对抗网络（GAN），通过生成器与判别器的对抗训练学习数据分布的高维映射。在此基础上，Mirza 等人提出的条件 GAN（cGAN）通过引入条件信息使生成过程可控。Isola 等人在 2017 年提出的 pix2pix 将 cGAN 构建为通用图像翻译框架，使用 U-Net 生成器 + PatchGAN 判别器，在标签图→街景、灰度图上色等任务上取得了突破。

在人脸素描合成领域，Fan 等人在 2022 年发布的 FS2K 数据集（2,104 对样本）是目前公开可用的最大规模数据集，涵盖三种素描风格、多种光照和面部属性，为深度学习方法的训练提供了关键数据支撑。

模型架构与技术细节

Model Architecture · pix2pix + U-Net + PatchGAN

Generator

U-Net（编码器-解码器）

  • · 8 层下采样 + 7 层上采样
  • · 跳跃连接保留边缘/轮廓细节
  • · Dropout(0.5) 防止过拟合
  • · 输入/输出: 256×256 RGB
Discriminator

PatchGAN（70×70 感受野）

  • · 4 层卷积下采样
  • · 对局部 patch 判别真伪
  • · InstanceNorm 保持样本独立
  • · 输出 30×30 判别图
Loss Function

对抗损失 + L1 损失（λ=100）

  • · GAN Loss: 交叉熵对抗训练
  • · L1 Loss: 逐像素结构约束
  • · Adam 优化器 (lr=2e-4)
  • · 后 50 epoch 线性衰减

▎跳跃连接核心实现

# models/networks.py — UnetSkipConnectionBlock.forward() def forward(self, x): if self.outermost: x = self.conv_in(x) # 下采样 x = self.submodule(x) # 递归中间层 x = self.conv_out(x) # 上采样 return x else: x_normal = self.conv_in(x) # 编码器输出（保留） x_mid = self.submodule(x_normal) x_up = self.conv_out(x_mid) # 解码器输出 # 关键：将编码器特征 x_normal 拼接到解码器输出 return torch.cat([x_up, x_normal], 1)

数据集与预处理管线

FS2K Dataset · 2,104 对配对样本

FS2K 数据集概览

  • · 人脸照片 2,104 张 + 专业手绘简笔画 2,104 张（一一配对）
  • · 三种素描风格（Style 1/2/3），多种光照、肤色、年龄段
  • · 属性标注：性别、微笑、耳环、发色、面部偏转角度等
  • · 训练集 1,058 对 / 测试集 1,046 对（~50:50）
  • · 实际有效配对样本：1,296 对

预处理流程

  1. 格式统一：全部转 JPEG + 3 通道 RGB
  2. 文件名对齐：photo/imageXXXX.jpg ↔ sketch/sketchXXXX.jpg
  3. 尺寸统一：双线性插值缩放至 256×256
  4. 左右拼接：简笔画（左）+ 人脸照片（右）→ 512×256
  5. 在线增强：随机水平翻转 (p=0.5) + 缩放裁剪 (286→256)

从"能用"到"好用"：两套改进方案

Beyond the Baseline · 解决手绘涂鸦生成模糊的核心痛点

核心痛点：基础 pix2pix 模型在 FS2K 的专业简笔画上表现良好，但用户随手画的粗糙涂鸦与训练数据存在明显的域偏移（Domain Gap）——线条不均匀、比例失调、风格迥异——导致生成人脸模糊甚至扭曲。针对这一问题，我设计了两套互补的改进方案。
方案一

检索增强模块：不改变生成模型，用"查找"弥补"生成"

特征提取 预训练 ResNet50 移除分类头 → 对 385 张真实人脸提取 2048 维特征向量
FAISS 索引 L2 距离索引 → 毫秒级相似度检索 → Top-3 准确率 82%
前端集成 Gradio 画板绘制 → 同时显示生成人脸 + 3 张最相似真实人脸（响应 < 0.1s）

核心理念：当用户手绘简笔画质量较差时，不强迫模型做"不可能的生成"，而是从真实人脸库中检索最匹配的结果作为候选。这是一种务实主义的工程思维——不追求完美生成，而是追求可用的用户体验。

方案二

数据增强 + 模型微调：让模型学会"看懂涂鸦"

模拟手绘数据增强 弹性变形 (α=50~150, σ=5~15) · 随机擦除 (2%~10%) · 形态学膨胀/腐蚀（改变线条粗细） · 椒盐噪声 (5%)
低学习率微调 加载预训练基础模型 → lr=2e-5（为初始的 1/10）→ 微调 30+30 epoch → G_L1 从 28.5 降至 21.0
量化提升 用户手绘测试集上：FID ↓ ~15% · SSIM ↑ ~28% · 主观生成效果明显改善

核心理念：与其让用户适应模型，不如让模型适应真实用户。通过数据增强模拟"粗糙涂鸦"的分布，并用低学习率微调防止遗忘已学到的专业简笔画知识，在不改变网络结构的前提下显著提升实用性。

▎多维可视化分析

特征图可视化 浅层响应边缘/轮廓 → 深层聚焦眼/鼻/嘴语义区域。验证了 U-Net 跳跃连接有效保留输入简笔画的结构信息。
Grad-CAM 注意力热力图 高亮区域集中于眼部、鼻部、嘴部 → 模型学会了将有限的简笔画线条映射到最具判别性的面部部件。
t-SNE 特征空间分布 真实人脸 vs 生成人脸形成两个相对分离的簇，几乎没有重叠 → FID 偏高（219.11）的直观解释：模型尚未完全学习真实人脸的复杂分布。
量化评估的坦诚结论

FID = 219.11（偏高，优秀模型通常 50~100），SSIM = 0.3365（偏低）。 主要原因：① 训练轮数仅 104 epoch，模型尚未充分收敛；② 256×256 分辨率限制了高频细节；③ 用户手绘与专业简笔画的域偏移。 但这正是课程设计的意义所在——不是追求 SOTA，而是完整地理解一个深度学习系统从数据到部署的全链路，并诚实地分析其局限性。

局限性与未来展望

Limitations & Future Work

当前局限

· 训练数据仅 2,104 对，模型参数量 5,400 万+，数据-容量不匹配

· 6GB 显存限制 batch_size=1 且分辨率仅 256×256

· 极度抽象简笔画（极少线条/比例失调）→ 生成可能扭曲

· 检索库仅 385 张，多样性有限

· FID 偏高（219.11），与 SOTA 存在差距

未来方向

· 引入扩散模型（Diffusion Models）替代 GAN，提升生成细腻度

· 联合 CelebA/FFHQ 等大规模人脸数据集扩充训练

· 知识蒸馏 + 量化 → 轻量化部署至移动端/Web

· 融入属性条件控制（性别/微笑/发色）→ 可控生成

· 优化检索库规模 + IVF/PQ 索引 → 保持毫秒级响应
